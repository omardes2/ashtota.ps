# قشطوطة بلبن — منصة الطلبات الإلكترونية 🍮

موقع طلبات إلكتروني متوافق مع الهاتف (Mobile-first, RTL) لمحل **قشطوطة بلبن**،
يتيح للزبائن تصفح المنتجات والطلب من أقرب فرع، مع بنية قابلة للتوسع لدعم عدد
غير محدود من الفروع.

> هذه هي **المرحلة الأولى (MVP – واجهة الزبون)**. الأقسام الأخرى (لوحة التحكم،
> شاشة الفرع، النقاط، السائقين، التقارير) مخطط لها في المراحل التالية — راجع
> [خارطة الطريق](#خارطة-الطريق).

---

## ✨ ما المنجز حاليًا

واجهة زبون كاملة تعمل بالكامل في المتصفح (بدون خادم) وتشمل:

- 🏬 **اختيار الفرع** — عرض حالة الفرع (مفتوح/مغلق) وأوقات العمل.
- 🍮 **المنيو حسب الفرع** — منتجات وأسعار وتوفّر مخصّص لكل فرع.
- 🗂 **تصنيفات** قابلة للتصفية (قشطوطة، بلبن، أم علي، كاسات، وافل...).
- ⚙️ **خيارات المنتج** — الحجم (إجباري)، الصوص، الإضافات، مع حساب السعر مباشرة.
- 🛒 **سلة مشتريات** كاملة (كمية، حذف، ملاحظات) محفوظة محليًا (localStorage).
- 🧾 **إتمام الطلب** — توصيل أو استلام، مناطق وأسعار توصيل، الحد الأدنى للطلب.
- 💬 **إرسال الطلب عبر واتساب** إلى رقم الفرع تلقائيًا (حل مبدئي قبل ربط API رسمي).
- 📱 **PWA** — قابل للتثبيت على الهاتف ويعمل جزئيًا دون اتصال.

## 🚀 التشغيل محليًا

الموقع ثابت (HTML/CSS/JS فقط) — لا يحتاج بناء. شغّل خادمًا بسيطًا:

```bash
# باستخدام Python
python3 -m http.server 8000
# ثم افتح: http://localhost:8000
```

أو افتح `index.html` مباشرة في المتصفح (بعض ميزات PWA تحتاج خادمًا).

---

## 📝 كيف تعدّل المحتوى؟

كل بيانات المنتجات والفروع حاليًا في ملف **`js/data.js`**. لتغيير:

- **رقم واتساب المحل**: عدّل `brand.whatsapp` (بصيغة دولية، مثل `970599123456`).
- **الفروع**: أضف/عدّل داخل مصفوفة `branches`.
- **المنتجات والأسعار**: مصفوفة `products` — لكل منتج قائمة `availability`
  تحدد الفروع المتوفر فيها وسعره في كل فرع.
- **مناطق التوصيل وأسعارها**: مصفوفة `deliveryZones`.

> لاحقًا (المرحلة الثانية) تُنقل هذه البيانات إلى قاعدة بيانات وتُدار من لوحة تحكم
> بدل تعديل الملف يدويًا.

### ⚠️ قبل النشر للعملاء

بدّل الأرقام والروابط الوهمية في `js/data.js` بالأرقام الحقيقية:
`whatsapp`, `phone`, وروابط `instagram/facebook/tiktok`.

## 🌐 النشر والربط مع GitHub + هوستنجر

راجع الدليل التفصيلي بالعربي: **[DEPLOY-HOSTINGER.md](DEPLOY-HOSTINGER.md)**

باختصار:
1. ارفع المشروع إلى مستودع GitHub (تم إعداده على هذا الفرع).
2. في لوحة hPanel بهوستنجر → **Git** → اربط المستودع.
3. عيّن مجلد النشر إلى `public_html` وحدّث تلقائيًا عند كل Push.

## 🗺 خارطة الطريق

### ✅ المرحلة 1 — واجهة الزبون (منجز)
اختيار الفرع، المنيو، الخيارات، السلة، إتمام الطلب، واتساب، PWA.

### 🔜 المرحلة 2 — الخادم ولوحة التحكم
- خلفية PHP + MySQL (متوافقة مع هوستنجر).
- لوحة تحكم الإدارة (فروع، منتجات، أسعار، طلبات).
- تسجيل الزبائن برقم الهاتف (OTP).
- شاشة استقبال الطلبات في الفرع + تنبيه صوتي.
- تخزين الطلبات وحالاتها بدل الاعتماد على واتساب فقط.

### 🔮 المرحلة 3 — ميزات متقدمة
نقاط الولاء، العضويات، الكوبونات، السائقين، التقارير، حملات واتساب الرسمية،
الدفع الإلكتروني، الإشعارات.

راجع "التصور المبدئي" الكامل الذي يوثّق كل الميزات المستهدفة.
