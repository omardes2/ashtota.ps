# قشطوطة بلبن — منصة الطلبات الإلكترونية 🍮

منصة طلبات حلويات لـ **قشطوطة بلبن**، بواجهة عصرية على طراز تطبيقات الطلب،
ولوحة تحكم إدارية كاملة — تعمل على استضافة هوستنجر المشتركة.

## 🧱 المعمارية

يتكوّن المشروع من جزأين يعملان معًا داخل `public_html`:

| الجزء | التقنية | المكان |
|------|---------|--------|
| **واجهة الزبون** | Next.js 14 + TypeScript + Tailwind (تصدير ثابت) | جذر المستودع (مبني) + المصدر في `frontend/` |
| **لوحة التحكم + الـ API** | PHP + SQLite | `admin/` و `api/` |

الواجهة تُبنى إلى ملفات ثابتة (HTML/CSS/JS) تُنشر في `public_html`، بينما
تبقى `api/` و `admin/` بلغة PHP تعملان على نفس الاستضافة.

---

## 📁 الهيكل

```
.
├── index.html, _next/, products/, product/, cart/ …   # واجهة Next.js المبنية (تُنشر)
├── api/            # الـ backend (PHP + SQLite): menu.php, order.php, admin/, install.php
├── admin/          # لوحة التحكم الإدارية (PHP)
├── service-worker.js   # يُنظّف كاش النسخة القديمة تلقائيًا
├── build-web.sh    # يبني Next ويصدّرها إلى الجذر
└── frontend/       # مصدر واجهة Next.js (للتطوير)
    ├── app/            # الصفحات (16 صفحة)
    ├── components/     # المكوّنات (layout, home, products, cart, branches, shared)
    ├── data/           # بيانات تجريبية (branches, categories, products, offers)
    ├── store/          # Zustand (cart, branch, wishlist, ui)
    ├── types/  lib/
    └── ...
```

## 🚀 التطوير والبناء

### تشغيل الواجهة محليًا (تطوير)
```bash
cd frontend
npm install
npm run dev        # http://localhost:3000
```

### البناء والنشر إلى الجذر (public_html)
```bash
bash build-web.sh  # يبني Next ويصدّرها إلى جذر المستودع بجانب api/ و admin/
```
ثم ادفع التغييرات إلى GitHub — وهوستنجر يسحبها إلى `public_html`.

> ملاحظة: هوستنجر لا يبني المشروع؛ لذلك نبني محليًا (`build-web.sh`) ونرفع الملفات
> المبنية. راجع `DEPLOY-HOSTINGER.md`.

## ✨ ميزات الواجهة

- 🏬 **اختيار الفرع الإجباري** عند أول زيارة (Modal) + تغيير الفرع مع تنبيه تفريغ السلة
- 🍮 16+ منتجًا، تصنيفات، بحث، فرز، فلترة، تخصيص (حجم/إضافات/ملاحظات)
- 🛒 سلة كاملة + كود خصم + إتمام طلب متعدد الأقسام + صفحة شكر + تتبع الطلب
- 📱 تصميم متجاوب مع Bottom Navigation للهاتف، RTL كامل
- 💾 حفظ الحالة في localStorage (Zustand)
- الصفحات: الرئيسية، المنتجات، تفاصيل المنتج، السلة، الدفع، نجاح الطلب، التتبع،
  العروض، الفروع، من نحن، التواصل، الدخول، إنشاء حساب، الحساب

## 🔌 الربط مع الـ Backend لاحقًا

الواجهة حاليًا تستخدم بيانات تجريبية في `frontend/data/`. لربطها بقاعدة البيانات
الحقيقية (لوحة التحكم)، استبدل قراءة البيانات بنداء `fetch("/api/menu.php")`
وإرسال الطلبات إلى `fetch("/api/order.php")` — الـ API جاهز ويعمل.

## 🗂 لوحة التحكم

`https://موقعك/admin/` — تُدار منها المنتجات والأسعار والفروع والطلبات.
التثبيت لأول مرة عبر `api/install.php`. التفاصيل في `DEPLOY-HOSTINGER.md`.
