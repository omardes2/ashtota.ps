# 📘 دليل ربط الموقع مع GitHub وهوستنجر (Hostinger)

هذا الدليل يشرح خطوة بخطوة كيف تربط موقع **قشطوطة بلبن** بمستودع GitHub، ثم
تربط GitHub بموقعك على هوستنجر لينشر تلقائيًا عند كل تحديث.

---

## الجزء الأول: رفع المشروع إلى GitHub

المشروع مرفوع بالفعل على GitHub في المستودع:

```
omardes2/ashtota.ps
```

على الفرع: `claude/qashoota-ordering-platform-bbj9ss`

### لدمج العمل في الفرع الرئيسي (main)
1. افتح المستودع على GitHub.
2. من صفحة الفرع اضغط **Compare & pull request**.
3. راجع التغييرات ثم اضغط **Merge** لدمجها في `main`.

> يُفضّل النشر على هوستنجر من فرع `main` بعد الدمج.

---

## الجزء الثاني: ربط GitHub بهوستنجر

هوستنجر يوفّر ميزة **Git** في لوحة hPanel تتيح سحب الكود مباشرة من GitHub.

### الطريقة (أ): عبر لوحة hPanel — الأسهل ✅

1. سجّل الدخول إلى **hPanel** على هوستنجر.
2. اختر موقعك (الاستضافة) → من القائمة الجانبية ابحث عن **Advanced → Git** (متقدم ← Git).
3. اضغط **Create a new repository / Create Repository**.
4. أدخل:
   - **Repository (رابط المستودع):**
     ```
     https://github.com/omardes2/ashtota.ps.git
     ```
     > إذا كان المستودع خاصًا (Private)، استخدم رابط SSH أو أضف
     > *Deploy Key* الذي يعرضه هوستنجر إلى إعدادات المستودع في GitHub
     > (Settings → Deploy keys → Add key).
   - **Branch (الفرع):** `main` (أو الفرع الذي تريد النشر منه).
   - **Directory (المجلد):** اتركه فارغًا أو ضع `public_html` لينشر في جذر الموقع.
5. اضغط **Create**. سيقوم هوستنجر بسحب الملفات.

### التحديث التلقائي عند كل Push (Webhook)

1. في نفس صفحة Git بهوستنجر، انسخ **Webhook URL** الظاهر.
2. اذهب إلى مستودعك في GitHub → **Settings → Webhooks → Add webhook**.
3. الصق الرابط في **Payload URL**، واختر:
   - **Content type:** `application/json`
   - **Which events:** *Just the push event*.
4. احفظ. الآن كل مرة تعمل فيها Push إلى الفرع، سيسحب هوستنجر التحديث تلقائيًا.

> بعض خطط هوستنجر تسحب التحديث تلقائيًا، وبعضها يتطلب الضغط على زر
> **Deploy / Pull** يدويًا من صفحة Git. تحقق من خطتك.

---

### الطريقة (ب): رفع يدوي عبر File Manager أو FTP (بديل بسيط)

إن لم تتوفر ميزة Git في خطتك:

1. حمّل ملفات المشروع كـ ZIP من GitHub (**Code → Download ZIP**).
2. في hPanel → **File Manager** → افتح مجلد `public_html`.
3. ارفع ملف ZIP و **استخرجه (Extract)** بحيث يكون `index.html` مباشرة داخل
   `public_html` (وليس داخل مجلد فرعي).

> عيب هذه الطريقة: يجب إعادة الرفع يدويًا عند كل تعديل.

---

## الجزء الثالث: التحقق من عمل الموقع

1. افتح نطاقك في المتصفح (مثال: `https://ashtota.ps`).
2. تأكد أن الصفحة الرئيسية تظهر بالعربية من اليمين لليسار.
3. جرّب: اختيار فرع → إضافة منتج → السلة → إتمام الطلب.

### ⚙️ ملاحظات مهمة

- **الملف الرئيسي** يجب أن يكون اسمه `index.html` وفي جذر `public_html`.
- إذا ظهرت صفحة قديمة، امسح الكاش (Cache) في المتصفح أو من hPanel.
- لتفعيل HTTPS: من hPanel → **SSL** → فعّل الشهادة المجانية لنطاقك.
- الموقع ثابت (Static)، لذا يعمل على **أي خطة هوستنجر** دون إعداد قاعدة بيانات.

## المرحلة الثانية: لوحة التحكم وقاعدة البيانات (PHP + SQLite)

تمت إضافة نظام خلفي كامل (Backend) يعمل مباشرة على هوستنجر **بدون إعداد قاعدة
بيانات MySQL** — يستخدم SQLite (ملف قاعدة بيانات تلقائي).

### التثبيت لأول مرة (خطوة واحدة)

1. تأكد أن الموقع منشور على هوستنجر (تم عبر Git).
2. افتح في المتصفح مرة واحدة:
   ```
   https://موقعك/api/install.php
   ```
3. ستظهر صفحة نجاح فيها **اسم المستخدم وكلمة مرور عشوائية** — **احفظها فورًا**
   (لن تظهر مرة أخرى).
4. اضغط "الدخول إلى لوحة التحكم" أو افتح:
   ```
   https://موقعك/admin/
   ```
5. سجّل الدخول، ثم من **الإعدادات** غيّر كلمة المرور.
6. **احذف ملف `api/install.php`** من File Manager (لأمان أكبر).

### لوحة التحكم تشمل

- 📊 لوحة قيادة (طلبات اليوم، المبيعات، حالات الطلبات)
- 🧾 الطلبات: عرض التفاصيل وتغيير الحالة
- 🍮 المنتجات: إضافة/تعديل/حذف + السعر والتوفر لكل فرع
- 🏬 الفروع، 🗂 التصنيفات، 🛵 مناطق التوصيل
- ⚙️ الإعدادات: اسم المتجر، واتساب، السوشال، النقاط، كلمة المرور

### أين تُخزَّن البيانات؟

في ملف SQLite **خارج مجلد الموقع** (`qashtoota_data/` بجانب `public_html`)،
فلا يصله أحد من الويب، ولا يُحذف عند تحديث الموقع من GitHub.

### 🗄️ التحويل إلى MySQL (قاعدة بيانات ظاهرة في hPanel)

إن رغبت بقاعدة بيانات MySQL تديرها من hPanel (phpMyAdmin):

**1. أنشئ قاعدة MySQL في هوستنجر**
- hPanel ← **Databases → MySQL Databases**.
- أنشئ قاعدة بيانات جديدة، واحفظ: اسم القاعدة، اسم المستخدم، كلمة المرور.
  (عادةً تكون بصيغة `uXXXXXXX_qashtoota`).

**2. أنشئ ملف الإعدادات السري على الخادم**
- في **File Manager** ← افتح مجلد `public_html/api/`.
- انسخ الملف `config.local.sample.php` وسمِّه **`config.local.php`**.
- عدّله وضع بيانات قاعدتك:
  ```php
  <?php
  define('DB_DRIVER', 'mysql');
  define('MYSQL_HOST', 'localhost');
  define('MYSQL_DB',   'uXXXXXXX_qashtoota');
  define('MYSQL_USER', 'uXXXXXXX_qashtoota');
  define('MYSQL_PASS', 'كلمة-المرور');
  ```
> 🔒 هذا الملف **غير مرفوع على GitHub** (كلمات المرور آمنة)، ولا يُحذف عند التحديث.

**3. ثبّت الجداول**
- افتح مرة واحدة: `https://موقعك/api/install.php`
- سيقول: `المحرك: mysql` ويُنشئ الجداول ويعطيك كلمة مرور مدير جديدة.
- الآن قاعدة بياناتك ظاهرة في **phpMyAdmin** وتديرها من hPanel.

> النظام اختُبر فعليًا على MySQL (إنشاء الجداول، عرض المنيو، حفظ الطلبات، لوحة التحكم) — كلها تعمل.

### ملاحظة أمنية

- بعد أول تثبيت، احذف `api/install.php`.
- غيّر كلمة مرور المدير من لوحة التحكم.
- المستودع عام، لذا لا توجد أي كلمات مرور داخل الكود (تُولَّد عشوائيًا عند التثبيت).
